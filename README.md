# Saber 11 Mathematics Score Modeling

This repository contains the code required to reproduce the analysis presented in the manuscript. The analysis uses Saber 11 mathematics scores from Colombia and fits Bayesian spatial mixed-effects models using INLA. The workflow includes data preprocessing, exploratory data analysis, spatial data processing, neighborhood graph construction, model fitting, model assessment, and generation of the reported tables and figures.

## Repository structure

```text
.
├── Data/
├── Data_Processing/
├── Descriptive/
├── Modelation/
├── Models/
└── README.md
```

## Data

The `Data/` folder contains the data objects used throughout the analysis. These include the spatial dataframes, the cleaned Saber 11 dataset, the modeling dataset, and the neighborhood graph used by the spatial model.

Large raw data files and intermediate objects may not be included in the repository. The objects in `Data/` can be regenerated by running the scripts in `Data_Processing/`, assuming the original Saber 11 data file is available.

The raw Saber 11 data are publicly available from the Colombian Institute for the Evaluation of Education (ICFES). To reproduce the analysis from scratch, download the corresponding Saber 11 dataset from ICFES and place it in the expected location inside the `Data/` folder with the name `Resultados_unicos_Saber_11_20260412.csv`. Then run the scripts in `Data_Processing/` to build the cleaned dataset, the spatial data objects, the modeling dataset, and the neighborhood graph.

## Data processing and graph construction

The `Data_Processing/` folder contains the scripts used to prepare the data for the analysis and construct the neighborhood graph required by the Besag spatial effect. These scripts perform the main preprocessing steps, including:

* filtering the Saber 11 data for the year used in the study
* recoding and translating categorical variables
* creating the parental academic level variable
* removing observations that cannot be matched to the spatial dataframe
* constructing the cleaned dataset
* constructing the final modeling dataset
* downloading the spatial dataframe required for the spatial model
* constructing the municipality-level neighborhood graph used in the Besag model

If the `Data/` folder is empty, run the scripts in `Data_Processing/` first. These scripts create the objects required by the descriptive analysis and model-fitting scripts.

## Descriptive analysis

The `Descriptive/` folder contains the exploratory data analysis used to motivate the selection of variables included in the final model. These scripts generate descriptive summaries and plots for the response variable and the selected covariates.

The descriptive analysis is used only to guide the construction of the fixed-effect component of the candidate models. The inferential results are obtained from the Bayesian models fitted using the script in the `Modelation/` folder.

## Model fitting and assessment

The `Modelation/` folder contains the script used for model fitting, model assessment, and posterior summaries. These scripts define the model formulas, fit the candidate INLA models, compute model comparison criteria, and generate posterior summaries.

The main fitted models include:

* a fixed-effects model
* a model with a municipality-level Besag spatial effect
* a model with both a municipality-level Besag spatial effect and a school-level iid random effect

The selected model includes fixed effects, a scaled Besag spatial effect for municipalities, and an iid school-level random effect.

## Fitted models

The `Models/` folder is used to store fitted model objects. This folder is intentionally empty in the GitHub repository because the fitted INLA model objects are large and cannot be uploaded efficiently.

To reproduce the fitted models, run the `Besag_Modeling.R` script in `Modelation/`. The scripts will save the fitted model objects locally in the `Models/` folder.

## Suggested order of execution

To reproduce the analysis from scratch, run the scripts in the following order:

1. Download the raw Saber 11 data (https://www.datos.gov.co/Educaci-n/Resultados-nicos-Saber-11/kgxf-xxbe/about_data) from ICFES and place it in the expected location inside `Data/` with the name `Resultados_unicos_Saber_11_20260412.csv`.
2. Run the scripts in `Data_Processing/` to construct the cleaned dataset, spatial dataframe, modeling dataset, and neighborhood graph.
3. Run the scripts in `Descriptive/` to reproduce the exploratory data analysis and descriptive figures.
4. Run the scripts in `Modelation/` to fit the candidate models, compute the model assessment criteria, and generate posterior summaries.
5. Use the generated objects and outputs to reproduce the tables and figures reported in the manuscript.

## Notes on reproducibility

The repository is organized so that the analysis can be reproduced from the public Saber 11 data and the scripts provided here. Some large intermediate objects and fitted models are not stored in the repository because of file-size limitations. These objects are generated automatically by running the corresponding scripts.

Users should check that the required R packages are installed before running the scripts. The analysis was carried out in R using packages for data manipulation, spatial data processing, and Bayesian model fitting with INLA.
